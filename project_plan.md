# ResNet50 目标定向对抗攻击与交互式 Web 演示系统开发计划

## 项目总览

本项目旨在基于PyTorch框架，利用本地计算资源（NVIDIA RTX 5060 GPU），对官方预训练的ResNet50分类图像模型进行安全性验证。核心目标不仅在于复现快速梯度符号法（FGSM）与投影梯度下降法（PGD）以实现目标定向的对抗样本攻击，最终还将**交付一个交互式的 Web 应用程序**。用户可通过该网站上传自定义图像，选择攻击算法（FGSM 单步攻击或 PGD 迭代攻击），通过可视化滑块动态调节扰动步长（$\epsilon$）与迭代次数，并实时观察对抗噪声叠加后，模型从高置信度的原始分类错误识别为指定目标分类的动态过程。

## 第一阶段：基线工程与底层模块化（Baseline & Modularization）

**状态**：已完成

**目标**：确保 GPU 加速正常运行，完成标准的前向传播推理，并将核心逻辑封装为便于 Web 端调用的模块。

- **任务 1.1：运行环境与算力验证**
  
  - **执行动作**：检查 Python 3.13 运行环境，确认已安装支持 CUDA 13.0 和 sm_120 架构的 PyTorch (Nightly) 版本，以消除 GPU 兼容性警告并释放 RTX 5060 的完整算力。

- **任务 1.2：构建模型加载与推理基准**
  
  - **核心逻辑**：加载 `ResNet50_Weights.DEFAULT` 预训练权重。动态获取 ImageNet-1K 标签映射表。
  - **代码规范**：将模型加载逻辑封装为 `AdversarialModel` 类，利用全局缓存机制（如 `@st.cache_resource`），确保在 Web 端多次推理时不会重复加载模型。
  - **基准测试**：遍历本地测试目录，输出原始预测的 Top-1/Top-5 类别及置信度（例：[dingo] (ID: 273) - 置信度: 38.95%），以此作为后续攻击的基准数据。

## 第二阶段：对抗攻击引擎开发（Attack Engine Implementation）

**状态**：已完成

**目标**：编写并封装实现 FGSM 与 PGD 目标定向攻击的核心数学逻辑，确保其能接受 Web 端的动态参数传入。

- **任务 2.1：攻击核心函数封装**
  
  - **执行动作**：引入 PyTorch 自动求导（Autograd）模块。确保输入图像张量设置 `requires_grad = True`。
  - 构建 `AttackEngine` 类，继承 `AdversarialModel`，提供 `prepare_adversarial_input`、`zero_gradients` 等基础方法。

- **任务 2.2：构建定向损失函数与梯度提取**
  
  - **核心逻辑**：将模型的输出与指定的目标类别 ID（Target ID，如 7 代表母鸡）进行交叉熵损失（Cross-Entropy Loss）计算。
  - **梯度反传**：调用 `loss.backward()` 获取输入图像数据域的梯度（Data Gradient）。

- **任务 2.3：FGSM 单步对抗样本生成**
  
  - **核心逻辑**：使用公式 $X_{adv} = X - \epsilon \cdot \text{sign}(\nabla_X J(X, Y_{target}))$ 生成对抗样本。
  - **参数接口**：将扰动步长参数 $\epsilon$ (Epsilon) 暴露为函数输入参数，用于接收 Web 端用户的滑动条输入。

- **任务 2.4：像素空间攻击重构（关键 Bug 修复）**
  
  - **问题发现**：在归一化空间直接执行 `torch.clamp(tensor, 0, 1)` 会严重扭曲扰动方向，导致边界像素梯度信息丢失；大 $\epsilon$ 下甚至出现置信度反弹（如 banana 置信度从 10% 回升至 21%）。
  - **修复方案**：将攻击下沉至像素空间 `[0, 1]`。在像素空间开启 `requires_grad=True`，反向传播得到 `grad_pixel`，执行 FGSM 扰动与合法截断后，再重新归一化供模型推理。由于乘除法均可微，Autograd 能将梯度顺畅地从归一化空间传播回像素空间。

- **任务 2.5：PGD 迭代攻击实现**
  
  - **核心逻辑**：基于 Madry et al. (ICLR 2018) 的 PGD 算法，实现多步迭代定向攻击。
  - 每步步长 $\alpha = \epsilon / 4$，每步后执行投影操作 $\Pi_\epsilon$（将对抗样本限制在与原始样本的 $L_\infty$ 距离不超过 $\epsilon$ 的邻域内），并截断到合法像素范围 `[0, 1]`。
  - **优势**：对高置信度、语义差距大的大类间攻击（如 banana → cock）成功率远高于单步 FGSM。

## 第三阶段：交互式 Web 网站开发（Interactive Web UI）

**目标**：基于 Streamlit 框架搭建前端网站，实现用户交互与对抗攻击过程的可视化，**同时支持 FGSM 与 PGD 两种攻击算法**。

- **任务 3.1：Web 页面布局设计**
  
  - **侧边栏控制区**：
    - 图片上传组件（支持本地图像上传）；
    - **攻击算法选择器**（FGSM 单步攻击 / PGD 迭代攻击）；
    - 目标类别下拉选择框（例如选择 Target ID 7 Hen）；
    - $\epsilon$ 值动态滑块（范围 0.00 ~ 0.10，步长 0.01）；
    - **PGD 专属控制**：迭代次数滑块（范围 5 ~ 50，步长 5，默认 20，仅在选择 PGD 时动态显示）。
  - **主展示区**：
    - 采用多列布局，并排展示"原始图像"、"提取的梯度噪声图"和"合成后的对抗样本图像"；
    - 当选择 PGD 时，可额外展示"迭代过程中目标类别置信度变化曲线"，直观呈现"逐步逼近"过程。

- **任务 3.2：状态同步与实时渲染**
  
  - **核心逻辑**：捕捉滑块/下拉框/算法选择器的变化事件，实时调用第二阶段的"对抗攻击引擎"。
  - **FGSM**：即时单步生成对抗张量并进行逆标准化（De-normalization）转为 RGB 像素值展示。
  - **PGD**：异步执行多步迭代，实时刷新当前迭代轮次与最新对抗样本预览；连续拖动滑块时需优化显存释放，防止 RTX 5060 OOM。

- **任务 3.3：置信度变化可视化**
  
  - **执行动作**：将原始样本和对抗样本分别送入 `eval()` 模式的 ResNet50 进行推理。
  - **预期输出**：
    - 使用柱状图（Bar Chart）直观对比原始分类（如 Dingo）置信度的显著下降，以及目标分类（如 Hen）置信度的跃升；
    - **PGD 专属**：折线图展示每轮迭代后目标类别置信度的上升曲线，直观对比单步 FGSM 与多步 PGD 的收敛差异。

- **任务 3.4：防御演示与一键加固**
  
  - 界面需支持"一键加固"（如高斯模糊、JPEG 压缩、对抗训练模型切换）；
  - 实时刷新防御后的识别准确度，并与攻击前对比。

## 第四阶段：防御与加固（Defense & Hardening）

**目标**：实现至少一种防御手段，量化防御前后模型对相同对抗样本的识别率变化。

- **任务 4.1：图像预处理防御**
  
  - 实现高斯模糊、中值滤波、JPEG 压缩等预处理手段，破坏对抗噪声的高频结构。
  - 对比防御前后，FGSM 与 PGD 对抗样本的 Top-1 识别率变化。

- **任务 4.2：对抗训练（Adversarial Training）**
  
  - 在训练过程中注入 FGSM/PGD 对抗样本，增强模型对扰动的鲁棒性。
  - 由于本项目使用预训练模型，可考虑在冻结 backbone 的基础上，对最后一层全连接层进行对抗微调。
